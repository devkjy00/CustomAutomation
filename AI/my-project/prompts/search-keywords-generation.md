# Search Keywords Generation Prompt

## Purpose
사용자 질문을 분석하여 웹 검색에 최적화된 검색어를 생성합니다.

## Template

```
사용자 질문: "{{query}}"

다음 기준에 따라 웹 검색에 최적화된 고품질 핵심 검색어를 생성하세요.

[핵심 목표]
사용자의 질문 의도를 파악하고, 검색 엔진이 가장 관련성 높은 결과를 반환할 수 있도록
불필요한 요소를 제거한 뒤 핵심 개념만 간결하게 정제된 검색어를 생성합니다.

[분석 지침]
1. 정보 요구(Intent Classification)
   - 질문이 요구하는 정보 유형을 파악하세요:
     • 최신 뉴스 / 사건
     • 가격 / 수치
     • 정의 / 개념 설명
     • 특정 인물 / 기업 / 제품 정보
     • 방법 / 튜토리얼
     • 비교 / 차이점
     • 위치 / 지역 기반 정보
   - 정보 유형에 따라 핵심 명사를 더 강조하거나 수식어를 포함합니다.

2. 핵심 개체 및 의미 단위 추출 (Entity Extraction)
   - 질문에서 주요 개체(인물명, 지명, 제품명, 기술용어 등)를 식별하세요.
   - 의미적으로 중요하지 않은 표현(감탄사, 말투, 요청 표현, 조사 등)은 제거합니다.
     예: "좀", "알려줘", "뭐야", "해줘", "지금", "혹시" 등

3. 검색 최적화용 키워드 구조화 (Keyword Structuring)
   - 핵심 명사 + 꼭 필요한 수식어 조합
   - 일반 명사는 가능한 구체명사로 치환 (예: "핸드폰" → "아이폰 16")
   - 정보 요구가 특정하면 해당 유형 키워드 포함
     예: 출시일, 가격, 스펙, 최신 뉴스, 원인, 해결법, 리뷰 등

4. 검색 효율성 고려
   - 2~5어절 사이로 유지 (너무 길면 검색 효율이 떨어짐)
   - 중복되는 의미 단위를 제거하고 가장 핵심 의미만 남깁니다.
   - 필요 시 다중 키워드를 공백으로 연속 배치하여 의미 밀도를 높입니다.

5. 언어 유지 규칙
   - 사용자 질문이 한국어면 한국어 검색어로 생성
   - 영어면 영어 검색어 생성
   - 언어 혼합 질문은 핵심 개체의 원어 표기 사용

6. 출력 규칙
   - 오직 검색어만 출력
   - 문장 형태·설명·불릿포인트·따옴표 금지
   - 키워드는 하나 또는 검색 성능 향상을 위해 최소 조합을 사용한 한 줄

[예시]
- "오늘 서울 날씨 어때?" → 서울 날씨
- "최신 AI 뉴스 알려줘" → 최신 AI 뉴스
- "삼성전자 주가가 얼마야?" → 삼성전자 주가
- "아이폰 16 언제 나오냐" → 아이폰 16 출시일
- "What is the latest news about ChatGPT?" → ChatGPT latest news
- "요즘 잘 나가는 전기차 뭐 있어?" → 전기차 인기 모델
- "Kafka와 RabbitMQ 차이 알려줘" → Kafka RabbitMQ 차이

출력: 최적화된 검색어 한 줄1
```

## Variables
- `{{query}}`: 사용자의 원본 질문

## Expected Output
최적화된 검색어 (문자열)
