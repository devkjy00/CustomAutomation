# Search Quality Verification Prompt

## Purpose
검색 결과가 사용자 질문에 충분히 답변할 수 있는지 엄격하게 평가합니다.

## Template

```
사용자 질문: "{{query}}"

검색 결과:
{{searchResults}}

아래 기준에 따라, 제공된 검색 결과가 사용자 질문을 해결하는 데 충분한지 평가하세요.

============================================================
[1단계: 질문 의도 분석]
다음 요소를 분석하여 사용자가 어떤 종류의 정보를 요구하는지 정의하세요.
- 정보 유형(Intent Type)
  • 사실/뉴스
  • 구체적 수치/값
  • 정의/설명
  • 절차/방법
  • 비교/차이
  • 목록/추천
- 요청한 정보가 “정답형(단일 값)”인지, “설명형(서술)”인지 구분하세요.

============================================================
[2단계: 검색 결과 평가]
검색 결과가 질문 의도와 **일치하는지**, 그리고 **실질적인 답**이 있는지 판단하세요.

다음 기준을 적용하세요:

1) **정확성(Answerability)**
   - 질문이 요구한 정보를 실제로 제공하는가?
   - 핵심 정보(숫자, 사건 내용, 정의 등)가 결과 내부에 존재하는가?

2) **관련성(Relevance)**
   - 검색 결과의 내용이 질문 주제와 직접적으로 관련되는가?
   - 제목만 있고 내용이 비어 있으면 “답변 불충분”으로 판단.

3) **완결성(Completeness)**
   - 질문이 요구한 정보를 만족할 만큼 충분한 구체성이 있는가?
   - 예:
     • “오늘 서울 온도는?” → 온도 값이 있어야 함
     • “최신 뉴스 보여줘” → 실제 사건 내용이 있어야 함
     • “hello 뜻이 뭐야?” → 정의가 있어야 함

============================================================
[3단계: 정량 기준 예시]

구체적 값을 요구하는 질문:
- 값이 없으면 → NO
- 값이 있으면 → YES

최신 뉴스 요구:
- 사건, 날짜, 내용이 포함되면 → YES
- “뉴스를 확인할 수 있습니다”처럼 일반 설명만 있으면 → NO

정의/설명 질문:
- 핵심 정의 제공 → YES
- 링크 설명 수준, 안내 문구만 존재 → NO

============================================================
[출력 형식]

hasAnswer: yes/no

missingInfo: 
- hasAnswer가 no일 경우, 어떤 정보가 부족한지 명확히 서술
- 한 문장 또는 핵심 키워드 중심 서술

연관 있는 검색 결과 본문 전체 
1...
2..
3..

예시:
hasAnswer: no
missingInfo: 질문이 요구한 '현재 온도 수치'가 검색 결과에 없음

============================================================

간결하지만 정확하게 출력하세요.
```

## Variables
- `{{query}}`: 사용자 질문
- `{{searchResults}}`: 검색 결과 목록 (제목 + 내용)

## Expected Output
```
hasAnswer: yes/no
missingInfo: 부족한 정보 설명
```
